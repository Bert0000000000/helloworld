# HelloWorld 技术架构文档

## 1. 系统架构

### 1.1 整体架构

```
┌─────────────┐     ┌─────────────┐
│   前端      │────▶│   后端      │
│ React +     │HTTP │   Flask     │
│ Ant Design  │◀────│   Python    │
└─────────────┘     └─────────────┘
```

### 1.2 技术选型

| 层级 | 技术 | 版本 | 说明 |
|------|------|------|------|
| 前端框架 | React | 18.x | 组件化开发 |
| UI 组件库 | Ant Design | 5.x | 企业级 UI |
| 构建工具 | Vite | 5.x | 快速构建 |
| 后端框架 | Flask | 3.0.0 | 轻量级 Web 框架 |
| 语言 | Python | 3.x | 后端开发语言 |

## 2. 后端设计

### 2.1 目录结构

```
backend/
├── app.py              # 主应用入口
└── requirements.txt    # 依赖配置
```

### 2.2 API 设计

#### 接口列表

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | / | 根路径欢迎信息 |
| GET | /api/hello | 问候语 API |
| GET | /api/status | 系统状态 API |

### 2.3 代码规范

- 遵循 PEP 8 编码规范
- 函数添加文档字符串
- 使用类型提示（可选）

## 3. 前端设计

### 3.1 目录结构

```
frontend/
├── src/
│   ├── App.jsx        # 主应用组件
│   └── main.jsx       # 入口文件
├── public/            # 静态资源
└── package.json       # 依赖配置
```

### 3.2 组件设计

**App 组件**
- 使用 Ant Design Layout 布局
- 集成 API 调用
- 响应式设计

### 3.3 状态管理

- 使用 React Hooks (useState, useEffect)
- 本地状态管理

## 4. 部署方案

### 4.1 开发环境

```bash
# 后端
cd backend && python app.py

# 前端
cd frontend && npm run dev
```

### 4.2 生产环境

**后端:**
- 使用 Gunicorn 作为 WSGI 服务器
- Nginx 反向代理

**前端:**
- 构建静态文件：`npm run build`
- 部署到静态服务器
